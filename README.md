# martini-saxs

Small command-line tool for estimating SAXS curves from Martini coarse-grained PDB structures.

This is a standalone CLI for people who are comfortable looking into the code when needed. It is not intended as a polished Python API. The main workflow is:

1. compute a SAXS curve from a Martini CG PDB
2. save the result bundle under an output base
3. generate a quick plot and peak summary from that result bundle

The tool supports two calculation modes:

- **Debye mode**: pairwise Debye summation over beads.  
  Best suited to **finite or isolated objects**, where a finite-object interpretation is appropriate.  
  An optional `--use-pbc-distances` flag is available to measure pair distances with minimum-image periodic boundary conditions, but this only changes the distance convention inside the Debye sum; it does **not** make the calculation equivalent to the periodic reciprocal-space treatment.

- **Periodic Fourier mode**: direct particle Fourier amplitudes on the reciprocal lattice.  
  Best suited to **periodic bulk systems**, where the simulation box is interpreted as a repeating unit cell and scattering is evaluated on the reciprocal lattice of that box.

It also includes an on-the-fly fallback for bead naming:

- if the input PDB already uses valid Martini bead types in `atom.name`, it is used directly
- otherwise, atom names are reconstructed from `.itp` residue definitions and a sibling `_updated.pdb` is written and used automatically

## Installation

Install directly from the repository main branch:

```
pip install "git+https://github.com/<user>/<repo>.git@main"
```

After installation, the CLI should be available as:

```
martini_saxs --help
```

If the command is not found, make sure the Python environment you installed into is the one currently active in your shell.

## Getting started.

Compute a SAXS curve

```
martini_saxs compute --pdb input.pdb
```

Everything else is set via a default and if no output is specified the results will be written according to the pattern

```
martini_saxs_<day>_<time>
```

The results can then be plotted with

```
martini_saxs plot
```

A file name has to be given to the plot, if the output was saved to a different naming scheme as the default.

## Arguments

### Compute command arguments

- `--pdb`  
  Path to the input PDB file.

- `--out-base`  
  Base output path. If omitted, a default `martini_saxs_<timestamp>` base name is generated.

- `--debye`  
  Use Debye summation instead of the periodic Fourier calculation.

- `--use-pbc-distances`  
  Only relevant together with `--debye`.  
  If set, Debye distances are computed with `box=dimensions` using minimum-image PBC distances.  
  If omitted, Debye distances use `box=None`.

- `--q-min`  
  Minimum q value in `1/nm`. Default: `0.0`

- `--q-max`  
  Maximum q value in `1/nm`. Default: `3.5`

- `--n-q`  
  Number of q points / bins. Default: `25`

- `--smooth-sigma-factor`  
  Optional Gaussian smoothing width for periodic Fourier mode, expressed in units of `dq`.  
  Example: `0.5` means `sigma_q = 0.5 * dq`

- `--max-workers`  
  Maximum worker count for Debye mode. Default: Python executor default.

- `--itp-dir`  
  Directory containing `.itp` files used for on-the-fly bead-name conversion. Default: `.`

- `--itp-pattern`  
  Glob pattern for `.itp` files. Default: `*.itp`

- `--quiet`  
  Reduce status output.

### Plot command arguments

- `--out-base`  
  Base output path used during compute.  
  If omitted, the tool searches for the latest `martini_saxs_<timestamp>` result.

- `--search-dir`  
  Directory to search when `--out-base` is omitted. Default: `.`

- `--peak-height`  
  Minimum peak height passed to `scipy.signal.find_peaks`. Default: `10.0`

- `--peak-max-q`  
  Only annotate peaks below this q value. Default: `3.5`

- `--title`  
  Plot title. Default: autogenerated from `out-base`

- `--x-min`, `--x-max`  
  X-axis limits. Defaults: `0.0`, `3.5`

- `--y-min`, `--y-max`  
  Optional y-axis limits. If one is set, both must be set.
